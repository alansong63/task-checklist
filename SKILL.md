# Task Checklist

长程任务的进度追踪工具，确保任务不会"断片"。

## 触发条件

**会触发：**
- "帮我做 xxx"
- "开始做 xxx"
- "创建个 checklist"
- "建个任务清单"
- "现在开始"

**不会触发：**
- 简单问答
- 查询类任务

## 核心功能

### 1. 创建 Checklist

根据任务类型自动生成，或用户自定义。

```
用户：帮我做个 xxx skill

→ 自动生成 checklist：
[ ] 1. 确认需求
[ ] 2. 写 SKILL.md
[ ] 3. 写核心代码
[ ] 4. 写文档
[ ] 5. Git 提交
```

### 2. 显示进度

随时查看当前进度：

```
当前进度：
[✓] 1. 确认需求
[ ] 2. 写 SKILL.md
[ ] 3. 写核心代码
[ ] 4. 写文档
[ ] 5. Git 提交

进度：1/5 (20%)
```

### 3. 更新状态

完成一步后更新：

```
用户：第2步完成了

→ 更新状态：
[✓] 1. 确认需求
[✓] 2. 写 SKILL.md
[ ] 3. 写核心代码
[ ] 4. 写文档
[ ] 5. Git 提交

进度：2/5 (40%)
```

### 4. 动态调整

可以添加、删除、调整步骤：

```
用户：再加一步测试

→ 更新 checklist：
[✓] 1. 确认需求
[✓] 2. 写 SKILL.md
[ ] 3. 写核心代码
[ ] 4. 写测试
[ ] 5. 写文档
[ ] 6. Git 提交
```

## 通用框架（必用）

所有任务使用这个通用框架：

```
[ ] 1. 需求定义 - 要做什么？
[ ] 2. 研发目标 - 做到什么程度？
[ ] 3. 执行计划 - 怎么做？
[ ] 4. 实施执行 - 具体实现
[ ] 5. 自检验证 - 做完了吗？
[ ] 6. 完成交付 - 结束
```

## 数据存储

文件位置：`~/.openclaw/workspace/.checklist/`

格式：
```json
{
  "task_id": "xxx",
  "title": "任务标题",
  "template": "skill-dev",
  "steps": [
    {"id": 1, "content": "确认需求", "status": "completed"},
    {"id": 2, "content": "写 SKILL.md", "status": "in_progress"},
    {"id": 3, "content": "写核心脚本", "status": "pending"}
  ]
}
```

## 与用户交互

### 创建时
1. 询问任务目标
2. 选择或自定义模板
3. 创建 checklist
4. 开始第一步

### 执行中
1. 每完成一步，用户告知
2. 自动更新状态
3. 显示新进度
4. 提示下一步

### 中断恢复
1. 检测是否有未完成任务
2. 询问是否继续
3. 恢复到断点
